## نظرة عامة

نظام شامل ومتكامل لإدارة مدارس تحفيظ القرآن الكريم، المتون العلمية، والقاعدة النورانية بشكل احترافي وذكي.

### الهدف الرئيسي

توفير منصة متكاملة تساعد في:
- إدارة المدارس والحلقات القرآنية بكفاءة عالية
- تقييم الطلاب بشكل ذكي ودقيق
- متابعة تقدم الطلاب في حفظ القرآن والمتون العلمية
- تسهيل التواصل بين جميع أطراف العملية التعليمية
- إصدار التقارير والشهادات بشكل احترافي

---

## المميزات الشاملة

### 1. إدارة شاملة متعددة المستويات

#### إدارة الجمعيات الخيرية
- **تسجيل جمعيات متعددة**: كل جمعية لها صفحتها المستقلة
- **إداريون مختصون**: لكل جمعية إداريون مع صلاحيات مخصصة
- **عزل البيانات**: كل جمعية تدير مدارسها وطلابها بشكل مستقل

#### إدارة المدارس
- **أنواع متعددة من المدارس**:
  - مدارس تحفيظ القرآن الكريم
  - مدارس المتون العلمية
  - مدارس القاعدة النورانية
  - مدارس مختلطة (كل ما سبق)

- **الفصول الدراسية**:
  - تقسيم حسب الأعمار (5-10، 11-15، 16-20، إلخ)
  - تقسيم حسب الجنس (ذكور، إناث، مختلط)
  - تقسيم حسب المستويات
  - تحديد عدد الطلاب المسموح في كل فصل

#### إدارة المستخدمين
- **7 أنواع من الأدوار**:
  1. مدير النظام (Super Admin) - صلاحيات كاملة
  2. إداري الجمعية (Organization Admin) - إدارة جمعية واحدة
  3. المشرف (Supervisor) - الإشراف على الحلقات
  4. المعلم (Teacher) - التدريس والتقييم
  5. المختبر (Examiner) - الاختبارات فقط
  6. ولي الأمر (Guardian) - متابعة الأبناء
  7. الطالب (Student) - الوصول المحدود

---

### 2. نظام القبول والتسجيل

#### تسجيل الطلاب
- **معلومات شاملة**:
  - البيانات الشخصية (الاسم، العمر، الجنس، الجنسية)
  - بيانات التواصل (البريد الإلكتروني، رقم الهاتف، واتساب)
  - بيانات ولي الأمر
  - المعلومات الصحية
  - المدرسة السابقة

- **رقم طالب فريد**: يتم إنشاء رقم فريد لكل طالب تلقائياً
  - صيغة: `STD-2024-000001`

- **حالات الطالب**:
  - قيد الانتظار (PENDING)
  - نشط (ACTIVE)
  - موقوف (SUSPENDED)
  - متخرج (GRADUATED)
  - منسحب (WITHDRAWN)

### 3. نظام التقييم الذكي

#### تقييم القرآن الكريم

##### أنواع التقييمات
1. **حفظ جديد (HIFZ_NEW)**: تقييم الحفظ الجديد
2. **مراجعة (MURAJA'A)**: مراجعة ما تم حفظه
3. **تسميع (TASMI'A)**: اختبار الحفظ
4. **إتقان (ITQAN)**: التأكد من إتقان الحفظ
5. **اختبار (EXAM)**: اختبار رسمي

##### معايير التقييم (0-100 لكل معيار)
1. **التجويد (Tajweed Score)**: 40% من الدرجة النهائية
   - أحكام التجويد
   - المخارج والصفات
   - المدود
   - الغنة
   - القلقلة
   - الإدغام والإخفاء

2. **الحفظ (Hifz Score)**: 40% من الدرجة النهائية
   - دقة الحفظ
   - عدد الأخطاء
   - السرعة

3. **الطلاقة (Fluency Score)**: 20% من الدرجة النهائية
   - السلاسة في القراءة
   - الوقف والابتداء

##### تسجيل الأخطاء بدقة
```typescript
{
  type: 'MAKHRAJ',        // نوع الخطأ
  ayah: 5,                // رقم الآية
  description: 'خطأ في مخرج حرف الضاد',
  severity: 'major'       // الخطورة: minor, moderate, major
}
```

##### أنواع أخطاء التجويد
- غنة (GHUNNAH)
- مد (MADD)
- قلقلة (QALQALAH)
- إدغام (IDGHAM)
- إخفاء (IKHFA)
- إقلاب (IQLAB)
- إظهار (IZHAR)
- تفخيم (TAFKHEEM)
- ترقيق (TARQEEQ)
- وقف (WAQF)
- مخرج حرف (MAKHRAJ)
- صفة حرف (SIFAH)

##### مستويات الحفظ التلقائية
بناءً على الدرجة الإجمالية:
- **ممتاز (EXCELLENT)**: 95% فأكثر
- **جيد جداً (VERY_GOOD)**: 85-94%
- **جيد (GOOD)**: 75-84%
- **مقبول (ACCEPTABLE)**: 60-74%
- **ضعيف (WEAK)**: 50-59%
- **يحتاج مراجعة (NEEDS_REVIEW)**: أقل من 50%

#### تقييم المتون العلمية

##### المتون المدعومة
1. **العقيدة (AQEEDAH)**
   - الأصول الثلاثة
   - العقيدة الواسطية
   - كتاب التوحيد

2. **الفقه (FIQH)**
   - عمدة الأحكام
   - بلوغ المرام
   - الأربعين النووية

3. **الحديث (HADITH)**
   - عمدة الأحكام
   - رياض الصالحين

4. **النحو (ARABIC)**
   - الآجرومية
   - قطر الندى
   - ألفية ابن مالك

##### معايير التقييم
1. **الدقة (Accuracy Score)**: 40%
2. **الطلاقة (Fluency Score)**: 30%
3. **الفهم (Understanding Score)**: 30%

#### تقييم القاعدة النورانية

- **17 درساً** من القاعدة النورانية
- معايير التقييم:
  1. **النطق (Pronunciation Score)**: 40%
  2. **التعرف (Recognition Score)**: 30%
  3. **القراءة (Reading Score)**: 30%

### 4. نظام الاختبارات الشامل

#### أنواع الاختبارات
1. **شفوي (ORAL)**: اختبار شفهي
2. **كتابي (WRITTEN)**: اختبار كتابي
3. **عملي (PRACTICAL)**: اختبار عملي
4. **مختلط (COMBINED)**: يجمع بين أكثر من نوع

#### المواد
- القرآن الكريم
- المتون العلمية
- القاعدة النورانية

#### إدارة الاختبارات
- **تحديد المختبرين**: تعيين مختبرين مختصين لكل اختبار
- **واجهة خاصة للمختبرين**: واجهة مخصصة لإدخال النتائج
- **تسجيل الدرجات**: تسجيل دقيق للدرجات مع التفاصيل
- **تحليل الأداء**: تحليل تلقائي لأداء الطلاب

### 5. نظام الرسائل المتعدد الأطراف

#### أنواع الرسائل المدعومة
1. **معلم ↔ طالب**
2. **معلم ↔ ولي أمر**
3. **إداري ↔ معلم**
4. **إداري ↔ طالب**
5. **إداري ↔ ولي أمر**

#### المميزات
- **صندوق وارد ذكي**: تنظيم الرسائل بشكل احترافي
- **علامة مقروء/غير مقروء**: تتبع حالة الرسائل
- **البحث المتقدم**: البحث في محتوى الرسائل
- **المرفقات**: إمكانية إرفاق ملفات
- **رسائل جماعية**: إرسال رسالة لعدة مستخدمين
- **رسائل لفصل كامل**: إرسال رسالة لجميع أولياء أمور طلاب فصل معين

#### التحكم في الصلاحيات
- فحص تلقائي للعلاقات بين المرسل والمستقبل
- منع الرسائل غير المصرح بها
- سجل كامل للرسائل

## التثبيت والإعداد

### المتطلبات الأساسية
```bash
- Node.js v18 أو أحدث
- PostgreSQL v14 أو أحدث
- Redis v6 أو أحدث (اختياري للتخزين المؤقت)
- npm أو yarn
```

### خطوات التثبيت

#### 1. استنساخ المشروع
```bash
cd yuoser
```

#### 2. تثبيت التبعيات
```bash
# تثبيت تبعيات الجذر
npm install

# تثبيت تبعيات Backend
cd backend
npm install

# تثبيت تبعيات Frontend
cd ../frontend
npm install
```

#### 3. إعداد قاعدة البيانات

##### إنشاء قاعدة البيانات
```sql
CREATE DATABASE quran_schools;
```

##### إعداد ملف البيئة
```bash
# في مجلد backend
cp .env.example .env
```

عدّل ملف `.env`:
```env
DATABASE_URL="postgresql://username:password@localhost:5432/quran_schools?schema=public"
JWT_SECRET=your_super_secret_key_here
```

##### تشغيل Migration
```bash
cd backend
npx prisma migrate dev
npx prisma generate
```

#### 4. إنشاء البيانات الأولية (Seed)
```bash
npm run seed
```

#### 5. تشغيل المشروع

##### بيئة التطوير
```bash
# من المجلد الجذر
npm run dev
```

سيعمل:
- Backend على: `http://localhost:5000`
- Frontend على: `http://localhost:3000`

##### بيئة الإنتاج
```bash
npm run build
npm start
```

## هيكل قاعدة البيانات

### الجداول الرئيسية

#### 1. المستخدمون والملفات الشخصية
- `User`: المستخدمون الأساسيون
- `Profile`: الملفات الشخصية
- `Teacher`, `Student`, `Guardian`, `Examiner`: جداول متخصصة للأدوار

#### 2. الجمعيات والمدارس
- `Organization`: الجمعيات الخيرية
- `OrganizationAdmin`: إداريو الجمعيات
- `School`: المدارس
- `Class`: الفصول الدراسية

#### 3. التقييم والتقدم
- `QuranProgress`: تقدم الطالب في القرآن
- `QuranEvaluation`: تقييمات القرآن
- `MutoonProgress`: تقدم الطالب في المتون
- `MutoonEvaluation`: تقييمات المتون
- `NooraniProgress`: تقدم الطالب في القاعدة النورانية
- `NooraniEvaluation`: تقييمات القاعدة النورانية

#### 4. الاختبارات
- `Exam`: الاختبارات
- `ExamResult`: نتائج الاختبارات
- `ExaminerAssignment`: تعيين المختبرين

#### 5. التواصل
- `Message`: الرسائل
- `Notification`: الإشعارات

#### 6. الحضور
- `Session`: الجلسات الدراسية
- `Attendance`: سجلات الحضور

#### 7. الشهادات
- `CertificateTemplate`: قوالب الشهادات
- `IssuedCertificate`: الشهادات الصادرة

#### 8. التدقيق
- `AuditLog`: سجل العمليات
- `SystemSettings`: إعدادات النظام

## أمثلة على الاستخدام

### مثال: إنشاء تقييم للقرآن الكريم

```typescript
const evaluation = await QuranEvaluationService.createEvaluation({
  studentId: 'student-uuid',
  teacherId: 'teacher-uuid',
  evaluationType: QuranEvaluationType.HIFZ_NEW,
  surah: 1,           // سورة الفاتحة
  fromAyah: 1,
  toAyah: 7,
  tajweedScore: 85,
  hifzScore: 90,
  fluencyScore: 88,
  tajweedErrors: [
    {
      type: TajweedErrorType.MADD,
      ayah: 4,
      description: 'مد زائد',
      severity: 'minor'
    }
  ],
  notes: 'أداء ممتاز، مع ملاحظة بسيطة في المد'
});
```

### مثال: إرسال رسالة

```typescript
const message = await MessagingService.sendMessage({
  senderId: 'teacher-uuid',
  receiverId: 'guardian-uuid',
  subject: 'تقرير أسبوعي',
  content: 'السلام عليكم، نود إعلامكم بتقدم ابنكم الممتاز هذا الأسبوع...',
});
```

### مثال: توليد تقرير شهري

```typescript
const report = await ReportsService.getMonthlyReport(
  'organization-uuid',
  new Date('2024-01-01')
);

// تصدير إلى Excel
const filepath = await ReportsService.exportToExcel('monthly', report);
```

## الخلاصة

هذا نظام متكامل وشامل تم بناؤه بأحدث التقنيات والمعايير البرمجية، مع التركيز على:

✅ **الشمولية**: يغطي جميع جوانب إدارة مدارس القرآن الكريم
✅ **المرونة**: قابل للتخصيص حسب احتياجات كل جمعية
✅ **الدقة**: نظام تقييم ذكي ودقيق
✅ **الأمان**: حماية كاملة للبيانات مع صلاحيات متعددة المستويات
✅ **سهولة الاستخدام**: واجهات بديهية وتجربة مستخدم ممتازة
✅ **القابلية للتوسع**: معماري ة قابلة للنمو
